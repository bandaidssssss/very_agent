专家经验Tips
目前rollout阶段峰值显存占用稳定在87%左右，计算log_prob的峰值显存99.53%，训练阶段反而在78%左右（TP 2, mbs 2, Zero 1, optimizer offload），但是同步模式目前还是存在大量的硬显存占用，尤其在显存限制下无法开启TP 2, mbs 2和TP 4, mbs 4的组合或者关掉optimizer offload。

有几个比较重要的专家经验：

actor TP不能随便改，这个牵一发动全身，容易爆prob或者训练显存，一般配合其他的一起改，SP通常需要打开
PP对整体性能的收益不高
log_prob mbs不宜开太大，这个通常是显存峰值。ref_prob mbs可以开大，这个阶段一般显存占用较小
actor model开param offload对整体性能影响很大，但是会导致rollout阶段显存占用极低，且无法很好利用，把rollout拉高也容易在log_prob阶段OOM
rollout TP可以减小甚至维持在1
max_num_batched_tokens不宜设置太大，反而会大幅降低rollout性能
rollout.n、学习率以及response_length截断率对算法性能影响很大
